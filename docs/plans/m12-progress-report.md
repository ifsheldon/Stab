# M12 Progress Report

## Milestone

M12: Performance Hardening

## Status

Partial progress, not milestone-complete.
This work starts the M12 measurement gate by making the primary benchmark matrix explicit, adding primary-baseline generation, adding release-profile compare report generation, recording row-level timing, variance, relative-ratio, paired submeasurement ratios, pass/fail status, and allocation metadata, adding beta-gate enforcement, adding profiler-note enforcement for rows slower than the hot-path threshold, adding source-owned profiler-note validation, adding regression-threshold enforcement infrastructure, adding beta memory-gate enforcement infrastructure, adding a source-owned primary memory baseline, promoting twelve faithful contract-representative rows to pinned public Stim CLI baselines, tightening direct-match benchmark gates to consider the worst paired submeasurement ratio, and optimizing the first sampler output, frame-reset, direct noisy Z-measurement, surface-sampler scalar and lazy packed-frame fast paths, gate lookup, sparse parser target storage, sparse XOR item toggling, result-format `01` reader, error-decomposition, probability-utility, stabilizer-string multiplication, and Pauli-string iterator benchmark paths.

## Contract

M12 requires a frozen primary benchmark matrix, a release-profile Stab-vs-pinned-Stim compare command, a report artifact with machine, compiler, Stim, Stab, benchmark-parameter, timing, variance, ratio, and status metadata, profiler notes for slow workloads, targeted optimizations behind existing abstractions, allocation tracking, regression thresholds, and all implemented oracle suites passing before and after performance changes.
This work covers only the matrix selection, primary-baseline selection, compare-reporting infrastructure, beta-gate enforcement, profiler-note gate infrastructure, source-owned profiler-note path validation, allocation-tracking report plumbing, source-owned primary memory-baseline validation, regression-threshold file enforcement, memory-gate compare-report enforcement, faithful public CLI baselines for the M8 primary repetition, primary rotated-surface, primary unrotated-surface, and high-repeat sample rows, M9 bit-packed m2d row, M9 primary detect and m2d rows, M10 high-repeat analyze_errors row, and M11 sparse, dense, repeated, and high-detector sample_dem rows, focused `CompiledSampler::sample_bytes` allocation, output-buffer, frame-reset, direct noisy Z-measurement, exact-generator measurement, direct H/CX operation, leading Z-reset elision, tiny-probability Pauli-channel, and lazy packed small-frame optimizations, gate lookup optimization, sparse parser target-storage and simple-instruction optimization, result-format `01` reader optimization, error-decomposition benchmark decomposition, direct probability-utility benchmark comparability, direct M6 PauliString and CliffordString multiplication benchmark comparability, and direct M6 PauliStringIterator benchmark comparability.
Optimization work across parser, sampler, detector, analyzer, and DEM hot paths remains pending M12 work, and the current milestone audit still records open under-specification issues before M12 can be considered complete.
