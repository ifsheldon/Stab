# Pinned Stim Qualification Adapter

This directory owns the narrow C++ adapter used by `stab-bench` when pinned Stim v1.16.0 has no faithful public CLI or `stim_perf` phase boundary.

The adapter is benchmark infrastructure only. It does not create a Stab C++ API or C++ header compatibility promise.

`ops/bench` validates the pinned Stim commit through a config-free Git view, materializes both committed Stab source and pinned Stim source into a fresh private `/tmp` runtime, and never reuses a prior CMake cache or library. It records canonical `cmake`, `cc`, `c++`, and `make` identities plus exact configure, library-build, compile, and environment contracts; reads CMake's generated `libstim` compile flags and reuses them for the standalone adapter so architecture-dependent Stim headers and the linked library agree; treats the pinned Stim include root as external through `-isystem` while retaining `-Wextra -Werror` for adapter-owned code; hashes the adapter call site, every isolated comparator source, and the resulting Stim library; recomputes the build fingerprint from all of those inputs; seals the final adapter executable; and rejects stale or internally inconsistent source, comparator, tool, library, binary, argument, environment, flag, or receipt identities before and after accepting output.

The adapter and Stab worker share a bounded schema-versioned JSON Lines protocol containing exact workload and measurement ids, evidence mode, iterations, semantic work, elapsed time, semantic output digest, setup and peak RSS, CPU affinity, pinned Stim commit, source digest, and build fingerprint. Contract preflight uses the distinct `contract` evidence mode and permits a finite zero elapsed duration because it is not statistical evidence; formal `timing` and `memory` rows still require finite positive elapsed duration.
Preparing any worker pair requires a canonical contract preflight covering protocol output, odd and even popcount and dense-XOR states, early-hit, all-zero, and late-hit `not_zero` states, sparse row-XOR and item-toggle states, allocating and in-place transpose states, odd and even Pauli and Clifford right-multiplication states, range and singleton Pauli-iterator states, direct DEM parse and canonical-print states, all accepted maxima, and every source-owned mandatory pre-barrier rejection.
Reports retain all 228 actual accepted row receipts or rejected process receipts and bind their recomputed digest into worker identity.
Offline replay reconstructs the source-owned receipt list and rejects omitted, reordered, altered, refingerprinted, or stale evidence.
The parent derives expected work from the selected source-owned scale, performs semantic preflight at the exact retained batch shape, and rejects malformed, oversized, duplicated, stale, non-finite, missing, unexpected, work-mismatched, or post-preflight digest-mismatched rows before computing any pair.

Timed qualification invocations use a one-byte start barrier. The parent pins the child and installs any regular-file limit before releasing that barrier, and both workers verify their singleton CPU affinity before measured work begins.

The PQ1 protocol-smoke body is deliberately synthetic and report-only. It proves runner symmetry and evidence reconstruction but is not a benchmark of Stim or Stab product behavior.

The first product adapter workload is `circuit-parse` with measurement `parse`. Both workers build the same bounded deterministic instruction fixture before timing, parse it for the calibrated iteration count, and consume an untimed digest of the resulting canonical circuit. The digest removes Stab's single terminal newline before comparison because pinned Stim's `Circuit::str()` omits that newline; it does not otherwise normalize circuit text. The probe proves worker symmetry but remains diagnostic. A promotable ratio additionally requires a source-owned runtime group, exact passing CQ preflight, a clean full or soak run, and a verified host.

The second product adapter workload is `circuit-canonical-print` with measurement `serialize`. Both workers build and parse the same exact fixture before the start barrier, then time only repeated conversion of the prepared circuit into canonical `.stim` text. Every produced string is consumed, the final string is digested outside timing, and only Stab's single terminal newline is removed before comparison. Parsing and fixture construction are setup work, while output allocation and destruction remain part of the serialization measurement.

The third product adapter workload is `gate-name-hash` with measurement `hash-all-names`. Both workers prepare the exact 82-entry Stim v1.16.0 gate table outside timing, reject partial table sweeps before the start barrier, and compare the accumulated hash checksum plus an order-sensitive table fingerprint.

The fourth product adapter workload is `simd-word-popcount` with measurement `toggle-popcount`. Both workers generate identical little-endian `splitmix64-word-v1` fixture words outside timing and require a 256-bit-aligned width from 512 through 268,435,456 bits. The Stim worker is compiled with the exact CMake-resolved `libstim` flags and reproduces pinned `simd_compat_popcnt` by toggling bit 300 and accumulating `ptr_simd[k].popcount()` across every architecture-dependent SIMD word, while Stab toggles the same bit and popcounts the complete `BitVec`. The exact call site in `main.cc` and loop in `simd_word_popcount_contract.h` are separately digest-bound through the generated inventory, runtime group contract, materialized adapter receipt, invocation, and report replay, so replacing the loop with a broader helper invalidates the contract even when a fixed output vector happens to match. The timed region contains only the symmetric compiler fence, toggle, popcount, and wrapping checksum accumulation. After timing, both workers digest eight little-endian `u64` fields in this exact order: checksum, iteration count, bit width, all four fixture-fingerprint lanes, and the final toggle bit as zero or one. The exact input bytes and digest, output digest, and semantic work must match before timing evidence is accepted. Every prepared worker pair exercises the shared frozen protocol vector, fixed odd and even popcount outputs, the accepted maximum, and all three pre-barrier rejection classes. The preflight digest also binds the exact source, build-fingerprint, and binary identities of both workers, so report replay rejects receipts transplanted from another pair; the standalone reproducibility command additionally proves two independent build identities. Setup and peak RSS remain report-only until PQ6 defines cross-scale memory acceptance.

The fifth product adapter workload is `simd-bits-xor` with measurement `xor-complete-vector`. Both workers generate the same two aligned vectors with `splitmix64-xor-pair-v1`, hash destination bytes followed by source bytes, and require a 256-bit-aligned width from 256 through 268,435,456 bits. Each timed iteration contains only the matching compiler fence and one complete-vector XOR; the Stab allocation-counter test requires zero timed allocations. After timing, both workers hash the complete final destination and unchanged source, then digest fourteen little-endian `u64` fields containing iterations, width, all four input lanes, all four destination lanes, and all four source lanes. Fixed odd, even, and accepted-maximum vectors plus all three pre-barrier rejection classes entered the 30-receipt fifth-slice preflight and remain present in the current 228-receipt preflight. `main.cc` and `simd_bits_xor_contract.h` are independently digest-bound, and the adapter receipt must contain the complete ordered set of compiled comparator sources while each runtime group selects its exact required subset.

The sixth product adapter surface is three `simd-bits-not-zero-*` workloads sharing measurement `not-zero`. The early workload reproduces pinned Stim's mislabeled `simd_bits_not_zero_100K` source at its actual 10,000-bit width and bit-600 hit. The all-zero and late-hit workloads force complete scans and receive independent group identities and 1.25x outcomes. All scales materialize exactly `ceil(bits / 64)` logical words while clearing every Stim padding word; widths from 64 through 268,435,456 are accepted without an artificial alignment rule. Each timed iteration contains only the matching compiler fence, an optimizer-opaque immutable input reference, one `not_zero` call, and Boolean checksum accumulation. The opaque reference prevents either compiler from replacing repeated scans with one precomputed result. Output binds checksum, iteration count, logical width, pattern marker, and all four input-digest lanes. The preflight freezes all three 10,000-bit patterns, the late accepted maximum, and both out-of-range rejection classes for both sealed workers. `main.cc` and `simd_bits_not_zero_contract.h` are independently digest-bound and remain present in current adapter receipt schema version 12.

The seventh product adapter surface contains independent `sparse-xor-row` / `row-xor` and `sparse-xor-item` / `xor-item` workloads. The row fixture reproduces pinned Stim's 1,000 sorted five-item rows and performs 999 forward plus 998 reverse symmetric differences per complete callback, so its qualification work denominator is 1,997 actual row XORs. The item fixture repeats exactly `2, 5, 9, 5, 3, 6, 10`, so one complete callback contains seven sorted insertion or removal toggles. Both workers execute two untimed complete callbacks to restore canonical state while retaining capacity, then time only complete callbacks behind matching compiler barriers and optimizer-opaque mutable references. Canonical little-endian input and final-state encodings bind exact odd, even, and accepted-maximum outputs, and partial-callback or over-cap counts are rejected before allocation and before the start barrier. `main.cc` and `sparse_xor_contract.h` are independently digest-bound through current adapter receipt schema version 12, and all historical sparse-XOR probes remain present in the current preflight.

The eighth product adapter surface contains independent `bit-matrix-transpose-in-place` / `transpose-square-in-place` and `bit-matrix-transpose-allocating` / `transpose-allocating` workloads. Both sides generate the same deterministic non-symmetric square matrix with eight deterministic set attempts per row, with duplicate columns collapsing to one set bit, require dimensions from 256 through 16,384 aligned to 256, and bind the canonical input plus final matrix bytes through exact digests. The in-place path executes two untimed transposes to restore canonical state before timing, then calls Stab's public `BitMatrix::transpose_square_in_place()` against pinned Stim's `simd_bit_table::do_square_transpose()` and requires zero timed Stab allocations. The allocating path executes and discards two untimed public transposes before timing, then calls Stab's public `BitMatrix::transpose()` against pinned Stim's `simd_bit_table::transposed()`, retains only the final result, and includes result allocation and prior-result destruction in the timed lifecycle. Small odd, small even, and accepted-maximum outputs plus every below-minimum, non-square, unaligned, over-cap, and valid-shape semantic-work-overflow rejection are frozen for both sealed workers; overflow must be rejected before fixture allocation and before consuming the enabled start barrier. `main.cc` and `bit_matrix_transpose_contract.h` are independently digest-bound through current adapter receipt schema version 12; current contract-preflight schema version 14 contains 228 accepted and rejected receipts, and current qualification report schema version 33 preserves those identities for replay.

The ninth product adapter surface is `pauli-string-right-multiply` / `right-multiply-in-place`. Both workers generate the same dense non-identity left and right operands from independently frozen SplitMix64 streams, preserve a positive left sign and negative right sign, execute two untimed complete multiplications to restore the canonical left state, then time the direct in-place public operation while accumulating every returned base-`i` exponent. The canonical input binds all four logical Pauli bit planes, while the 17-field output binds iteration count, checked semantic work, width, workload marker, phase checksum, final left state, and unchanged right state. The runtime scales are 10,000, 100,000, and 1,000,000 qubits with an accepted maximum of 1,048,576. Current adapter receipt schema version 12 and contract-preflight schema version 14 retain the odd, even, maximum, zero-width, over-cap, wrong-measurement, and semantic-work-overflow receipts for both workers. The inherited identity-only perf callbacks remain legacy provenance and cannot qualify this full-width mutation contract.

The tenth product adapter surface contains independent `pauli-iter-range` and `pauli-iter-singleton` workloads sharing measurement `construct-and-iterate-borrowed`. Both workers construct one public iterator per callback, traverse every value through borrowed `iter_next`, consume output count and result width, and destroy the iterator before returning. The range group enumerates X/Z weights 2 through 5 at widths 5, 11, and 22; the singleton group enumerates X/Y/Z weight 1 at widths 1,000, 32,000, and 1,000,000. Range width 22 is the accepted private 1,000,000-output cap and width 23 is rejected; singleton width 1,048,576 is accepted at the public Pauli limit and width 1,048,577 is rejected before allocation or barrier consumption. The groups stay separate because their advancement algorithms and scale behavior differ. `main.cc` and `pauli_string_iter_contract.h` are independently digest-bound through current adapter receipt schema version 12; contract-preflight schema version 14 contains 228 accepted and rejected receipts, and current qualification report schema version 33 preserves those identities for replay.

The eleventh product adapter surface contains independent `clifford-string-right-multiply-identity` / `right-multiply-identity` and `clifford-string-right-multiply-non-identity` / `right-multiply-non-identity` workloads. Both use equal-width public in-place multiplication at 10,000, 100,000, and 1,000,000 qubits, with an accepted maximum of 1,048,576. The identity contract preserves the exact pinned upstream identity-by-identity workload, while the non-identity contract repeats Stim's canonical 24-gate order against all 23 non-identity right operands in a complete 552-position cycle. A checked 64-byte descriptor binds width, workload marker, fixture schema, gate count, right-cycle count, complete span, public cap, and reserved zero. Both workers prepare operands and an independent scalar expectation before the barrier; final states use scalar group exponentiation, and non-identity witnesses advance one precomputed width stride per observed lane so setup does not replay `iterations * width` products. They reset callback and result-witness state immediately before the barrier, then time only public in-place multiplication behind matching compiler fences and optimizer-opaque references. The 16-field SHA-256 output binds work, both final operands, callback count, and a witness read from the mutated left operand. The identity group uses source-owned independent-throughput timing with a smaller-count exact common semantic batch and normalized seconds-per-work ratios because its Stim and Stab stable timing ranges do not overlap; the non-identity group and every other group retain common iterations. The 72 Clifford receipts contain 36 requests per worker: ten accepted vectors and 26 rejected vectors covering every field-specific or work-overflow rejection plus opposite valid workload markers, width-to-work mismatch, and malformed descriptor hex in Stab-then-Stim order. The exact legacy 10K measurement pair was mapped only to the identity `small` runtime scale before its completion-authorized migration. Adapter receipt schema version 12, contract-preflight schema version 14, and report schema version 33 reject omitted, reordered, modified, or stale Clifford evidence.

The twelfth product adapter surface contains independent `dem-parse` / `parse` and `dem-canonical-print` / `serialize` workloads. Both workers generate the exact heterogeneous eight-item DEM cycle outside timing at 64, 4,096, and 65,536 top-level items; the accepted contract maximum is 524,288 items. Parsing times repeated construction and replacement of an owned model, while printing parses one setup model and times repeated owned-string production. Both workers consume every result, retain the final result, and compare exact canonical bytes after normalizing only the known single terminal-newline difference. The sixteen mandatory DEM receipts cover small odd, small even, accepted maximum, and first-over-cap rejection for each worker and method; separately selected adapter probes cover zero, incomplete-cycle, wrong-measurement, and semantic-work-overflow guards. `main.cc` and `dem_model_contract.h` are independently digest-bound through adapter receipt schema version 12, contract-preflight schema version 14, and report schema version 33.
